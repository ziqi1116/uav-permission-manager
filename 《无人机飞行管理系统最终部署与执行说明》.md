# 无人机飞行管理系统最终部署与执行说明

---

## 第一章 项目概述

### 1.1 系统名称

**无人机飞行权限管理与审批系统**（无人机飞行管理系统）

### 1.2 技术架构

- **基础框架**：RuoYi v3.9.1
- **后端**：Spring Boot + MyBatis + 若依安全与权限体系
- **前端**：Vue + Element UI + Vue Router + Axios
- **前后端分离**：后端提供 REST API，前端独立部署；登录态为 Token（JWT）
- **数据库**：MySQL 8.0（InnoDB，utf8mb4）
- **地图**：高德地图 JavaScript API（监控轨迹与违规点展示）

### 1.3 主要功能模块

- 首页（统计大屏，支持按角色展示全部/个人数据）
- 无人机管理（CRUD，按 owner_id 数据权限）
- 飞行申请（申请、编辑/删除仅待审批且本人）
- 审批管理（管理员/审批角色）
- 飞行权限（只读，按 applicant_id 过滤）
- 监控管理（多任务轨迹、回放、违规点标记）
- 违规管理（按 permission 归属过滤）
- 系统管理（仅管理员）

## 第二章 菜单结构说明

### 2.1 最终一级菜单结构（扁平化）

所有业务菜单为**一级菜单**（parent_id=0），无“无人机飞行管理”父级；首页置顶。

| 序号 | 菜单名称   | 路由 path  | 说明                     |
|------|------------|------------|--------------------------|
| 0    | 首页       | index      | 统计大屏（个人/全部）    |
| 1    | 无人机管理 | droneInfo  | 无人机 CRUD              |
| 2    | 飞行申请   | application| 申请列表与新增/编辑/删除 |
| 3    | 审批管理   | approve    | 待审批列表与通过/驳回    |
| 4    | 飞行权限   | permission | 权限列表（只读）         |
| 5    | 监控管理   | monitor    | 多任务轨迹与回放         |
| 6    | 违规管理   | violation  | 违规记录                 |
| -    | 系统管理   | system/*   | 若依原有系统管理目录     |

**已删除/不展示**：若依官网、统计报表（独立菜单，首页即统计）、无人机飞行管理（父级目录）。

## 第三章 权限设计说明

### 3.1 admin（超级管理员）

- **范围**：全部菜单、全部按钮、全部数据。
- **实现**：user_id=1 或 role_key=admin，不做数据范围过滤；拥有所有 `sys_role_menu`（含系统管理、审批、违规等）。

### 3.2 drone_user（无人机用户）

- **范围**：仅业务相关菜单与按钮，无审批、违规、系统管理。
- **分配菜单**：首页、无人机管理、飞行申请、飞行权限、监控管理。
- **不分配**：审批管理、违规管理、系统管理、统计报表独立菜单。
- **按钮**：无人机 list/query/add/edit/remove；飞行申请 list/query/add/edit/remove；飞行权限 list/query；监控 list/query（无 export 等扩展）。

### 3.3 数据级权限控制

- **无人机**：非 admin 仅能看/改/删 `owner_id = 当前用户` 的数据。
- **飞行申请**：非 admin 仅能看 `applicant_id = 当前用户`；编辑/删除仅本人且 status=待审批。
- **飞行权限**：非 admin 仅能看 `applicant_id = 当前用户`（通过 application 关联）。
- **监控**：任务列表仅 `applicant_id = 当前用户`；轨迹/违规点接口校验 permission 归属当前用户。
- **违规记录**：非 admin 仅能看与本人 permission 关联的违规。
- **首页统计**：非 admin 仅统计 `applicant_id = 当前用户`。

## 第四章 数据权限说明

### 4.1 Service 层过滤逻辑

- 列表/导出：在查询前若非 admin，则设置 `entity.setOwnerId(getUserId())` 或 `entity.setApplicantId(getUserId())`，或通过 `params.applicantId` 传入 Mapper。
- Mapper XML：使用 `<if test="ownerId != null">AND owner_id = #{ownerId}</if>` 或 `params.applicantId`、`applicant_id` 条件，保证 SQL 仅查当前用户数据。

### 4.2 防越权机制

- **详情/编辑/删除**：先按 ID 查询，再校验 `owner_id` 或 `applicant_id` 是否等于当前用户；非本人则返回“无权限”或 403。
- **监控轨迹/违规点**：按 permissionId 查询前，先查该 permission 的 applicant_id，非本人则抛“无权限查看该飞行任务数据”。

### 4.3 统计接口数据隔离

- 首页 dashboard 及图表接口：非 admin 调用时传入 `applicantId = getUserId()`，统计 SQL 中增加 `applicant_id = #{applicantId}`（或通过 permission 关联违规表），保证普通用户仅看到个人统计。

## 第五章 监控轨迹模块说明

### 5.1 多任务

- 左侧任务列表来自接口：`GET /drone/monitor/task/list`（分页）；非 admin 仅返回 `applicant_id = 当前用户` 的任务。
- 点击某任务后，请求该任务的轨迹与违规点，地图只加载当前选中任务数据，不一次加载全部。

### 5.2 轨迹回放

- 播放/暂停、进度条（或步进索引）；使用单次 `setTimeout` 链驱动动画，不在未清理的 `setInterval` 中跑。
- 在 `beforeDestroy` 中清除回放定时器、地图覆盖物（polyline、marker、circle）及地图引用，避免内存泄漏。

### 5.3 违规标记

- 违规点接口：`GET /drone/monitor/violation/{permissionId}`，返回带经纬度的违规记录。
- 地图上以红色 marker 标出；轨迹点中 `is_violation=1` 可在 polyline 或 marker 上做区分（如标红）。

### 5.4 地图实现逻辑

- 使用高德地图；通过 `loadAMap.js` 统一加载脚本，避免重复引入。
- 绘制：polyline（轨迹）、当前点 marker、违规点红色 marker；可选 circle 表示批准范围；`fitView()` 自适应视野。
- 切换任务时先清除旧 polyline/marker/circle，再加载新轨迹与违规点；仅加载当前选中任务的轨迹与违规点。

## 第六章 执行步骤

**本系统仅保留以下可执行 SQL（均在 `sql/` 目录）：**

| 顺序 | 文件 | 说明 |
|------|------|------|
| 1 | `ry_20250522.sql` | 若依基础初始化（用户/角色/菜单/部门等） |
| 2 | `drone_flight.sql` | 业务表与业务菜单（无人机、申请、审批、权限、监控、违规、统计等） |
| 3 | `drone_system_final_install.sql` | 最终整合：菜单扁平化、首页与角色权限、监控表增强、模拟数据、测试用户 user1 |

其他零散 SQL 已合并进上述文件，无需再执行。

### 6.1 导入若依基础 SQL

- 执行 **`sql/ry_20250522.sql`**，完成 `sys_user`、`sys_role`、`sys_menu`、部门岗位等表结构与初始 admin 数据。

### 6.2 执行业务表与菜单 SQL

- 执行 **`sql/drone_flight.sql`**，创建 `drone_info`、`flight_application`、`flight_permission`、`flight_monitor`、`violation_record` 及字典、业务菜单。

### 6.3 执行最终整合安装脚本

- 执行 **`sql/drone_system_final_install.sql`**。
- 该脚本将：菜单结构修复（删除若依官网、统计报表、无人机飞行管理父级并扁平化）、角色 drone_user 创建与菜单分配、首页权限修复、监控表字段增强、模拟测试数据（3 条任务、每条约 20 条轨迹、含违规点）、测试用户 user1 及绑定 drone_user。
- 若已执行过监控表 ALTER，重复执行可能报“Duplicate column”，可注释脚本中“五、监控表结构增强”整段。

### 6.4 启动后端

- 使用 IDE 或 `mvn spring-boot:run` 启动 `ruoyi-admin`；确认配置中数据库、Redis、端口与前端一致。

### 6.5 启动前端

- 在 `ruoyi-ui` 目录执行 `npm install`、`npm run dev`；如需地图，在 `.env` 中配置 `VUE_APP_AMAP_KEY`。

### 6.6 登录测试

- 使用 **admin** 登录，确认能看到全部菜单、全部数据及监控多任务轨迹。
- 使用 **user1** 登录，确认仅有首页、无人机管理、飞行申请、飞行权限、监控管理；首页统计为个人数据；监控仅本人任务；审批/违规/系统管理不可见且接口 403。

## 第八章 常见问题排查

### 8.1 首页 403 或“没有权限”

- **原因**：drone_user 未分配首页菜单或首页菜单 `parent_id`/`status`/`visible` 异常。
- **处理**：执行 `drone_system_final_install.sql` 中“四、首页权限修复”相关语句；或执行 `fix_home_permission.sql`。确认 `sys_role_menu` 中存在 `role_id=drone_user` 且 `menu_id=首页`。后端 dashboard 接口已支持 `index:view`、`drone:statistics:view`、`drone:statistics:query` 任一即可访问。
- **建议**：退出登录 → 清理浏览器缓存 → 重新登录，使菜单与权限重新加载。

### 8.2 菜单不显示

- **原因**：角色未分配对应菜单，或菜单 `status`/`visible` 异常。
- **处理**：在系统管理 → 角色管理中检查 drone_user 是否勾选首页、无人机管理、飞行申请、飞行权限、监控管理；或重新执行最终脚本中的“三、菜单权限分配”和“四、首页权限修复”。admin 应拥有全部菜单（若依默认或通过角色分配）。

### 8.3 地图加载失败

- **原因**：未配置高德 Key 或 Key 不可用、域名未加入白名单。
- **处理**：在 `ruoyi-ui` 的 `.env` 中配置 `VUE_APP_AMAP_KEY`；在高德控制台检查 Key 与 Referer/域名白名单。未配置时前端应有“请在 .env 中配置 VUE_APP_AMAP_KEY”的提示。

### 8.4 权限未刷新

- **原因**：菜单或角色权限已改，但前端仍用旧 Token 或缓存。
- **处理**：退出登录 → 清理浏览器缓存（或无痕模式）→ 重新登录。必要时后端清理在线用户缓存（如 `DELETE FROM sys_user_online`，视若依版本而定），再让用户重新登录。
