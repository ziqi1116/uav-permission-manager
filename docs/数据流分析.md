## 二、认证与权限数据流

### 2.1 登录

| 步骤 | 位置 | 数据流 |
|------|------|--------|
| 1 | 前端 `views/login.vue` | 用户输入 username、password、验证码 code + uuid，调用 `api/login.js` → `login()` |
| 2 | 请求 | `POST /dev-api/login`（代理到 `http://localhost:8011/login`），Body: username, password, code, uuid |
| 3 | 后端 | `SysLoginController.login()` → 校验验证码(Redis) → 校验用户密码 → 生成 JWT token → 写入 Redis(LoginUser) |
| 4 | 响应 | 返回 `{ token, ... }` |
| 5 | 前端 | `store/modules/user.js` → `Login()` → `setToken(res.token)` → 跳转 `/` 或 redirect |

### 2.2 获取用户信息与路由（进入系统后）

| 步骤 | 位置 | 数据流 |
|------|------|--------|
| 1 | 前端 `permission.js` | 路由守卫：有 token 且 `roles.length === 0` → `GetInfo()` |
| 2 | 请求 | `GET /dev-api/getInfo`，Header: `Authorization: Bearer <token>` |
| 3 | 后端 | TokenService 解析 token → 从 Redis 取 LoginUser → 返回 user、roles、permissions |
| 4 | 前端 | `store/user` 写入 id、name、roles、permissions；随后 `GenerateRoutes()` |
| 5 | 请求 | `GET /dev-api/getRouters` |
| 6 | 后端 | `SysLoginController.getRouters()` → `SecurityUtils.getUserId()` → `menuService.selectMenuTreeByUserId(userId)` → `menuService.buildMenus(menus)` |
| 7 | 数据来源 | `sys_menu` + `sys_role_menu`：按当前用户角色过滤菜单，构建树 + 组件路径(Layout/views/xxx) |
| 8 | 前端 | `store/modules/permission.js` → `filterAsyncRouter(res.data)` → `loadView(component)` → `router.addRoutes(rewriteRoutes)`，再 `addRoutes(asyncRoutes)`、404 兜底 |

之后所有业务请求均在 Header 中携带 `Authorization: Bearer <token>`，后端通过 `SecurityUtils.getUserId()` / `getUsername()` 做身份与数据权限控制。

## 三、核心业务数据流

### 3.1 业务实体与表关系

```
sys_user (若依用户)
    │
    ├── drone_info (owner_id)              无人机信息
    │
    ├── flight_application (applicant_id)  飞行申请
    │         │
    │         ├── flight_permission (application_id)  飞行权限（审批通过后生成）
    │         │         │
    │         │         ├── flight_monitor (permission_id)  飞行监控/轨迹点
    │         │         └── violation_record (permission_id)  违规记录
    │         │
    │         └── 审批人 approve_user → sys_user
```

### 3.2 无人机管理 (droneInfo)

| 方向 | 数据流 |
|------|--------|
| 列表 | 前端 `listDroneInfo(query)` → `GET /drone/droneInfo/list` → `DroneInfoController.list()` → 非管理员带 `applicantId` 数据权限 → `DroneInfoMapper.selectDroneInfoList()` → `drone_info` 表 |
| 新增/修改/删除 | 前端表单/表格 → `POST/PUT/DELETE /drone/droneInfo/*` → Controller → Service → Mapper → `drone_info` 增删改 |

### 3.3 飞行申请 (application)

| 方向 | 数据流 |
|------|--------|
| 列表 | 前端 `listApplication(query)` → `GET /drone/application/list` → `FlightApplicationController.list()` → `applyDataScope(非管理员限 applicantId)` → `FlightApplicationMapper.selectFlightApplicationList()` → `flight_application` 联表 sys_user、drone_info |
| 新增 | 前端 `addApplication(form)` → `POST /drone/application`，Body 不含 applicantId → Controller 内 `setApplicantId(getUserId())`、`setCreateBy(getUsername())` → `FlightApplicationServiceImpl.insertFlightApplication()` → 写 `flight_application`；申请单号可由 `GET /drone/application/genApplicationNo` 预生成 |
| 修改/删除 | 非管理员仅允许本人、待审批状态；Controller 内校验 → Service 更新/删除 `flight_application` |

### 3.4 审批 (approve) —— 核心联动流

| 步骤 | 数据流 |
|------|--------|
| 1 列表 | 前端 `listApprove(query)` → `GET /drone/approve/list` → 仅待审批或按角色过滤 → `flight_application` |
| 2 通过 | 前端 `approve(id, remark)` → `POST /drone/approve/approve/{id}` → `FlightApproveController` → `IFlightApplicationService.approve(applicationId, getUserId(), approveRemark)` |
| 3 业务 | **FlightApplicationServiceImpl.approve()**（事务）：① 更新 `flight_application` 状态=已通过、审批人、审批时间、备注；② **插入 flight_permission**（approved_area/height/start/end、permission_status=有效）；③ **插入 flight_monitor** 一条初始记录（permission_id 关联） |
| 4 驳回 | `POST /drone/approve/reject/{id}` → 仅更新 `flight_application` 状态=已驳回、审批意见，不生成 permission/monitor |

```
申请(待审批) ──通过──► 更新 application ──► 新增 permission ──► 新增 monitor 初始记录
            ──驳回──► 仅更新 application 状态
```

### 3.5 飞行权限 (permission)

| 方向 | 数据流 |
|------|--------|
| 列表 | 前端 `listPermission(query)` → `GET /drone/permission/list` → 数据权限(非管理员限 applicantId 通过 application 关联) → `FlightPermissionMapper` 联表 `flight_application`、`sys_user` 等 → 只读展示 |

权限记录仅由“审批通过”生成，本模块无新增入口。

### 3.6 监控管理 (monitor) —— 轨迹与违规

| 接口 | 数据流 |
|------|--------|
| 任务列表 | 前端 `getTaskList()` → `GET /drone/monitor/task/list` → 分页、非管理员限 applicantId → `FlightMonitorMapper.selectTaskListPage()` → 有效/进行中的 permission + 申请信息 |
| 轨迹 | 前端 `getTrack(permissionId)` → `GET /drone/monitor/track/{permissionId}` → `FlightMonitorMapper.selectTrackByPermissionId()` → `flight_monitor` 表 longitude、latitude、flight_time 等，按 flight_time 升序 |
| 违规点 | 前端 `getViolationPoints(permissionId)` → `GET /drone/monitor/violation/{permissionId}` → 查 `violation_record` 按 permission_id |
| 前端展示 | 高德地图：轨迹 Polyline(path)、起点/终点/违规点 Marker；地图中心 setCenter(起点)、setZoom(15) |

监控表 `flight_monitor` 可被其他上报服务按 permission_id 写入轨迹点；违规由 `violation_record` 表记录并在此展示。

### 3.7 违规管理 (violation)

| 方向 | 数据流 |
|------|--------|
| 列表/详情 | 前端 → `GET /drone/violation/list`、`GET /drone/violation/{id}` → 数据权限 → `ViolationRecordMapper` + 关联 permission/application |
| 新增/修改/处理 | 前端 → `POST/PUT /drone/violation` → Controller → Service → Mapper → `violation_record` 表（含 longitude、latitude 等） |

### 3.8 首页统计 (statistics / dashboard)

| 接口 | 数据流 |
|------|--------|
| 仪表盘 | 前端 `getDashboard(begin, end)` → `GET /drone/statistics/dashboard` → 权限 `index:view` 或 `drone:statistics:query`；非管理员传 `applicantId = getUserId()` |
| 后端 | `IFlightStatisticsService.selectDashboardStats(applicantId)` → 今日申请数、今日通过数、当前飞行任务数、累计违规数；另 `selectApplicationCountByDate`、`selectAreaUsage`、`selectViolationCountByDate` 供图表 |
| 数据来源 | `flight_application`、`flight_permission`、`violation_record` 等聚合/按日统计 |

## 五、关键数据流小结

1. **认证**：Login → Token(Redis) → 后续请求带 Token → getInfo/getRouters 决定角色与菜单。
2. **申请→权限→监控**：用户提交申请 → 审批通过时**一条事务内**写 application 更新、permission 插入、monitor 初始插入；监控页按 permission_id 查轨迹与违规点。
3. **数据权限**：非管理员在 application、permission、monitor 任务列表、statistics 等处通过 `applicantId = getUserId()` 或 params 传入，在 Mapper XML 中加 `AND applicant_id = #{applicantId}` 或等价条件。
4. **前端路由**：getRouters 返回菜单树 → 转成 Vue Router 配置(含 Layout、views 下组件) → addRoutes 注册；监控管理等新页面依赖该动态路由正确注册。

以上即为本项目的整体数据流分析，可作为开发、联调与答辩说明依据。
