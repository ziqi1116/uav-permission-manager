## 第一章 项目概述

无人机飞行管理系统基于 **RuoYi v3.9.1** 进行二次开发，采用 **前后端分离架构**：

- **后端技术栈**：Spring Boot、Spring Security、JWT、MyBatis、MySQL、Redis。
- **前端技术栈**：Vue（RuoYi-Vue）、Vue Router、Vuex、Element-UI、Axios。
- **业务特性**：无人机台账管理、飞行申请与审批、飞行权限管理、飞行监控与轨迹回放、违规告警、统计分析。

## 第二章 安装步骤

1. **导入若依原始 SQL**
   - 在 MySQL 中执行官方初始化脚本（示例文件名：`ry_20250522.sql`），完成 RuoYi 基础表结构与基础数据初始化。
2. **执行项目整合脚本 `drone_system_final_install.sql`**
   - 在同一数据库中执行 `sql/drone_system_final_install.sql`：
     - 创建本项目新增业务表（无人机、申请、权限、监控、违规）。
     - 初始化业务字典、角色、菜单与菜单权限。
     - 扁平化无人机业务菜单，删除「无人机飞行管理」「统计报表」「若依官网」等多余菜单。
     - 写入模拟数据（3 条任务，每条 20 条轨迹，含违规点位）和测试账号 `user1`。
     - 开启注册开关（`sys.account.registerUser = true`），支持前端自助注册。
3. **启动后端**
   - 进入 `ruoyi-admin` 模块，使用 Maven 或 IDE 启动 Spring Boot 应用。
   - 默认端口参考项目配置（如 `8080`），确保连接到上述已初始化的数据库与 Redis。
4. **启动前端**
   - 进入 `ruoyi-ui` 目录，安装依赖并启动：
     - `npm install`
     - `npm run dev`
   - 默认访问地址：`http://localhost:9527`。

## 第三章 角色说明

- **admin**
  - 超级管理员，拥有系统全部菜单与数据权限，可管理用户、角色、菜单、参数配置等。
- **drone_user**
  - 无人机业务角色，仅拥有如下业务菜单：
    - `首页`
    - `无人机管理`
    - `飞行申请`
    - `飞行权限`
    - `监控管理`
  - 仅能操作与本人相关的数据，受数据权限控制，无法越权查看其他用户的数据。

## 第四章 注册功能说明

- **前端注册入口**
  - 登录页面底部右侧展示文案：**「没有账号？去注册」**，点击跳转至 `/register` 注册页。
  - 注册页表单字段：
    - 用户名：登录账号，需唯一。
    - 昵称：展示名称，用于系统中显示。
    - 密码 / 确认密码：长度 5–20，包含基础强度校验，需两次一致。
    - 验证码：复用 RuoYi 图形验证码接口 `/captchaImage`。
- **后端注册接口**
  - 地址：`POST /register`
  - 控制器：`SysRegisterController`，调用 `SysRegisterService.register`。
  - 核心流程：
    1. 校验系统是否开启注册（`sys.account.registerUser` 为 `true`）。
    2. 校验验证码是否正确且未过期（Redis 验证）。
    3. 校验用户名非空、密码非空、长度合法、账号唯一。
    4. 使用 **BCryptPasswordEncoder** 对密码进行加密（通过 `SecurityUtils.encryptPassword`）。
    5. 初始化用户状态为启用（`status = '0'`），昵称来自注册表单。
    6. 持久化用户信息。
    7. 成功后自动为新用户绑定 `drone_user` 角色。
  - 注册成功后：
    - 前端弹出成功提示，并自动跳转回登录页。
    - 用户使用新注册账号密码登录后，将以 `drone_user` 角色身份使用系统。

## 第五章 数据权限说明

- **admin**
  - 不受数据权限限制，可查看与操作全部用户及业务数据。
- **非 admin（包括 drone_user 及其他业务角色）**
  - 通过 Service 层的 `@DataScope` / 数据权限逻辑进行过滤：
    - 用户只能看到自己或本部门范围内的数据（依据角色 `data_scope` 配置）。
    - 无人机、飞行申请、飞行权限、监控、违规记录等查询接口均集成相应数据范围过滤。
  - 防越权控制：
    - Service 层统一调用数据权限检查方法，访问不在授权范围内的数据会抛出异常。
    - 后端只返回当前登录用户有权限查看的数据，无论前端如何构造参数都无法突破。

## 第六章 监控轨迹说明

- **不规则轨迹**
  - 后端监控表 `flight_monitor` 存储经纬度、速度、高度、时间、是否违规等字段。
  - 前端使用高德地图 `AMap.Polyline` 绘制轨迹，支持多拐点、不规则折线、地球曲线（`geodesic`）等效果。
- **自动居中**
  - 点击某个飞行任务后：
    - 清空旧轨迹及 Marker。
    - 加载该任务对应权限下的轨迹点列表。
    - 以任务起点 `[lng, lat]` 为中心调用 `map.setCenter`，并设置合适缩放级别（如 15 级）。
    - 同时渲染起点 / 终点标记，保证当前任务位置清晰可见。
- **多任务切换**
  - 每次切换任务前调用 `map.clearMap()` 清理上一条任务的 Polyline 与 Marker，避免图层叠加。
  - 使用统一方法加载并渲染轨迹，保证切换时不会重复初始化地图实例，提升性能。
- **违规标记**
  - 违规记录表 `violation_record` 存储违规类型与经纬度。
  - 前端以红色 Marker 单独标记违规点，并可在 Hover/点击时展示违规类型与说明，便于监管人员快速定位问题点。

## 第八章 常见问题

1. **首页 403 / 无法访问**
   - 检查是否已执行 `drone_system_final_install.sql`。
   - 确认 `sys_menu` 中存在 `path = 'index'` 且 `parent_id = 0` 的菜单。
   - 确认 `sys_role_menu` 中为对应角色（admin / drone_user）分配了首页菜单。
2. **地图加载失败**
   - 检查前端是否正确配置高德地图 JS SDK Key。
   - 确认浏览器控制台无跨域或脚本加载错误。
   - 确认后端监控与违规接口返回的数据经纬度不为空且顺序为 `[lng, lat]`。
3. **菜单不显示**
   - 对于 `user1` 或新注册用户：
     - 确认 `sys_user_role` 中已绑定 `drone_user` 角色。
     - 确认 `drone_system_final_install.sql` 中关于 `drone_user` 的菜单分配语句已执行（可重复执行脚本修复）。
   - 若仅显示首页：
     - 清理浏览器缓存，重新登录。
     - 确认后端 `/getRouters` 返回的路由数据已包含无人机相关菜单。
4. **注册失败**
   - 检查验证码是否正确、是否超时（验证码由 Redis 管理，使用一次即失效）。
   - 检查系统参数 `sys.account.registerUser` 是否为 `true`（可在参数管理中查看或通过脚本更新）。
   - 确认用户名未被占用，密码长度 5–20，未包含非法字符。
